Cured Hosting Diagnostics — What this package does

This package improves reliability, security, performance, and debugging for Cured Hosting customers. It contains modular components that work together to reduce downtime and speed root-cause analysis.

Components

  • PlagueDr Diagnostics — Core runtime safety engine capturing preflight snapshots, intercepting fatal crashes and allowing safe rollbacks.
  • PlagueDr SEO — SEO helpers, Open Graph defaults, and optional advanced schema with an admin UI.
  • Media Pipeline — Background media processing hooks to prepare optimized responsive media.
  • Sentinel — Security monitoring and checks for suspicious activity.
  • Copilot / Code Assistant — Developer utilities to speed diagnostics and fixes.
  • Airbag — Protective runtime guard rails for risky operations.
  • Beak Armor / PlagueDr Core — Shared bootstrapping and utilities.
  • Leech Drain Cache — Safe cache management helpers.
  • Optional modules — Gallery, banner, affiliate modules loaded conditionally to avoid fatal activations.

Why this helps Cured Hosting

  • Less downtime via automatic detection and rollback of fatal plugin activations.
  • Faster debugging with telemetry and captured crash logs.
  • Improved performance through media optimization and cache helpers.
  • Production safety with guard rails to avoid destructive changes.
  • Built-in SEO readiness for social previews and search schema.
  • Modular design that won't fail when optional pieces are absent.

Installation & Operational Notes

  • Install as a regular plugin; avoid activating both MU and plugins copies simultaneously.
  • Check server PHP error logs and the `plaguedr_resolved_clash_log` option if you see a fatal error.
  • Premium features are gated by license flags and will return `LOCKED` when not active.
