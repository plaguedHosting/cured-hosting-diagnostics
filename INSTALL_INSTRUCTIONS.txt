License token behavior:
- The plugin now stores license tokens in a hashed form (`pd_license_token_hash`) and uses an explicit boolean flag (`pd_is_pro`) to enable premium features. On first run, if an existing legacy raw token is found it will be migrated automatically to the new hashed storage.
- To reset the license (for troubleshooting), use the admin Cleanup button on Settings > PlagueDr License, or delete the following options via WP-CLI or the database: `pd_is_pro`, `pd_license_token_hash`, and `pd_license_token` (legacy).

Admin token generator:
- The plugin provides an admin-only "Generate Owner Token" button on Settings > PlagueDr License. When clicked it will:
  - Create a one-time raw token and display it in the UI (copy it immediately).
  - Store a hashed verification value in `pd_license_token_hash` and set `pd_is_pro = 1`.
  - Schedule any pro-only cron tasks.
  - The raw token is stored only in a short-lived transient and is not kept in plaintext in the DB.
